Intelligram for Instagram

An iOS app leveraging machine learning to analyze Instagram photos and predict engagement based on image quality patterns. This project reached 50,000+ users and analyzed 3+ million photos, but taught me that technical success doesn't overcome founder misalignment.

I share this project to highlight a few things: working within constraints of existing ML tools, building features around technical capabilities, achieving real user adoption and value, and learning that technical success doesn't overcome founder misalignment.

The Genesis

Co-founder Aram pitched the concept during lunch, having discovered an ML model capable of analyzing social media photos for technical quality while predicting user engagement. Rather than building our own model, we decided to work within constraints by developing features around this existing tool.

The project involved creating an iOS app leveraging an existing machine learning model trained on social media images. The model could assess image quality and predict engagement likelihood based on patterns from its training data. Using Apple's Core ML for on-device processing meant no server calls were required, ensuring complete privacy. The core function provided real-time image quality analysis with engagement prediction.

Technical Approach

  • Framework: Apple's Core ML for on-device processing
  • Privacy: No server calls required
  • Core Function: Real-time image quality analysis with engagement prediction

Features We Built

We developed several features that extended beyond the core ML functionality:

  • Journey tracking showing improvement over time
  • Anonymous Wall for peer feedback
  • Category-specific predictions (selfies, landscapes, food, animals)
  • One-tap Instagram integration

Results & Outcomes

The app reached 50,000+ users and analyzed 3+ million photos. Notable adoption occurred in Colombia and Russia with approximately 25,000 downloads, and users reported engagement improvements.

Aram pursued the project through Dublin Institute of Technology's New Frontiers programme with Enterprise Ireland funding. The co-founder relationship ultimately proved insufficient for sustained collaboration post-funding.

Good idea, real users, genuine value—yet technical capability alone cannot overcome founder misalignment.

Key Learnings

This project taught me several important lessons about product development and startup dynamics:

  • Working within constraints: We didn't need to build our own ML model. Finding an existing tool and building features around it was faster and more practical.
  • Privacy by design: Using on-device processing with Core ML meant we could offer complete privacy without server calls, which became a key differentiator.
  • Real user value: The app achieved genuine adoption with 50,000+ users and meaningful engagement improvements reported by users.
  • Founder alignment matters: Despite technical success and user adoption, founder misalignment ultimately prevented sustained collaboration. This was perhaps the most important lesson—technical capability alone isn't enough.

Conclusion

Intelligram demonstrated that you can build meaningful products by working within the constraints of existing tools rather than building everything from scratch. We achieved real user adoption and delivered genuine value. However, the project also highlighted that technical success and user traction aren't sufficient if the founding team isn't aligned on vision, execution, and long-term goals.

The experience reinforced my belief in pragmatic product development—finding the right tools, building features that matter, and ensuring team alignment from the start. Sometimes the most important lessons come from projects that succeed technically but fail organizationally.
